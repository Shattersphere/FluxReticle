# Project Doc Index

- `.agent/BRIEF.md`: read first for current state, commands, live target, blockers, and next action.
- `AGENTS.md`: repo rules, command pointers, deploy policy, and verification policy.
- `README.md`: user-facing overview and feature summary.
- `HANDOVER.md`: stable project facts, ownership map, and validation notes.
- `PLANS.md`: active work and blocked items only.
- `.agent/archive/2026-05-12-initial-fork-setup.md`: completed setup/refactor history moved out of the active plan.

## When To Read

- Build/deploy/config/rendering work: read `.agent/BRIEF.md`, then `AGENTS.md`.
- Ambiguous or multi-step feature work: read `PLANS.md`.
- Runtime debugging or handoff: read `HANDOVER.md`.
- User-facing explanation or package overview: read `README.md`.
- Historical context: read archive notes only when the active docs are insufficient.
